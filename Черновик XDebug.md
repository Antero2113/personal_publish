# XDebug

**XDebug** — расширение PHP, упрощающее отладку кода. Позволяет задавать брейкпоинты, останавливая исполнение кода, просматривать текущие значения суперглобальных переменных PHP ($GLOBALS, $_GET, $_POST, $_FILES, $_REQUEST, $_SESSION, $_ENV, $_COOKIE, $_SERVER), пользовательских переменных, стек вызовов.

Это значит, что можно перемещать брейкпоинты, пока не обнаружится проблемный участок, не добавляя логирование в код.

## Установка XDebug

Здесь и далее примеры команд, конфигурации будут ориентированы на версию php 7.4, которая используется в наших проектах. 

Команда установки для ОС Linux:

```
sudo apt-get install php7.4-xdebug
```

### Конфигурационный файл 

Располагать конфигурационные параметры расширения можно в отдельном файле ```/etc/php/7.4/mods-available/xdebug.ini``` или в ```php.ini```.

Копируем и вставляем этот код в файл конфигурации:

```
[xdebug]
xdebug.mode=debug
xdebug.start_with_request=yes
xdebug.client_host=127.0.0.1
xdebug.client_port=9003
xdebug.log=/var/log/xdebug.log
```

### VSCode Extension

Следующий этап — установка расширения в VS Code для работы с XDebug. 

Можно скачать по ссылке:
```
https://marketplace.visualstudio.com/items?itemName=xdebug.php-debug
```

Или перейти в VS Code → Extensions, ввести в поиске «PHP Debug».

### Настройка в проекте

1. Открыть проект в VS Code.
2. Перейти к Run and Debug panel (Ctrl+Shift+D).
3. Клик на иконку settings рядом с зелёным треугольником или создать launch.json файл:

```
{
   "version": "0.2.0",
    "configurations": [
        {
            "name": "Listen for XDebug",
            "type": "php",
            "request": "launch",
            "port": 9003
        },
        {
            "name": "Launch currently open script",
            "type": "php",
            "request": "launch",
            "program": "${file}",
            "cwd": "${fileDirname}",
            "port": 9003
        }
    ]
}
```
4. Нажать ``start debugging`` (зелёный треугольник).
5. Запускаем проект, как обычно.

## Возможности отладки

### Брейкпоинты

Устанавливаются нажатием перед номером строки, на которой нужно задержать выполнение: http://shots.m18.ru/iwsxt8m8l0/

Когда все breakpoints установлены, можно запускать исполнение скрипта или проект целиком на localhost, дебаггер будет останавливаться на указанных точках и выделять их в коде: http://shots.m18.ru/tcbvb7upr8/.

Слева в самом низу есть настройка брейкпоинтов http://shots.m18.ru/i2efp46xj2/: 
- Кроме добавленных пользователем точек, XDebug останавливается на ошибках, предупреждениях и т.п. (можно отметить нужное)
- В правом верхнем углу этого окошка можно добавить свою функцию для брейкпоинтов, удалить все добавленные ранее, запустить отладку.

#### Условия для брейкпоинтов

В VS Code можно установить условные брейкпоинты — они срабатывают только при выполнении определённого условия.

Установка: клик правой кнопкой по точке брейкпоинта → Edit Breakpoint → ввести условие.

### Оценка выражений (Watch)

В разделе WATCH (слева) можно добавлять произвольные выражения для отслеживания. Например:

- ```$user->getName()```
- ```count($items)```
- ```strlen($buffer)```

Это позволяет наблюдать за вычисляемыми значениями.

2. Слева сверху можно просматривать значения всех переменных: http://shots.m18.ru/gdwoe570at/.

### Управление процессом исполнения кода

Основная панелька для управления отладкой: http://shots.m18.ru/t29ubf40no/:

1. Continue/Pause (F5)
2. Step Over (F10) — выполняется текущая строка кода. Если в ней присутствует вызов функции, отладчик не погружается в её реализацию — функция выполняется целиком «за кулисами», после чего управление передаётся следующей строке в текущем файле.
3. Step Into (F11) — при наличии вызова функции на текущей строке отладчик переходит внутрь неё, позволяя пошагово проследить исполнение кода внутри этой функции. Если вызовов на строке нет, поведение идентично Step Over — выполняется текущая строка, и отладчик переходит к следующей.
4. Step Out (Shift+F11) — выполнение текущей функции доводится до конца без остановок внутри неё, после чего отладчик возвращается на ту строку, откуда эта функция была вызвана. 
5. Restart (Ctrl+Shift+F5)
6. Stop (Shift+F5)
